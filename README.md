## üéØ Mission
When a child is taken across international borders, the left-behind parent enters a chaotic world of conflicting laws (Hague Convention), law enforcement gaps, and emotional trauma.

**Recovery Hub** replaces scattered notebooks and panic with a centralized, intelligent dashboard. It helps parents:
1.  **Organize** their legal strategy.
2.  **Document** evidence securely.
3.  **Communicate** effectively with authorities using AI-assisted drafting.

## ‚ú® Key Features

### üõ°Ô∏è Crisis Management
*   **Intelligence Dossier:** AI analyzes the specific country pair (e.g., USA -> Japan) to determine Hague Convention compliance and risk factors.
*   **Universal Task List:** Auto-generates a priority checklist based on the specific legal situation (Custody Order vs. No Order).
*   **Live Strategy Guide:** A real-time, voice-activated AI companion (using Gemini Live API) to help ground parents and plan next steps during high-stress moments.
*   **Strategy Brainstormer:** A chat interface to discuss concerns and instantly generate actionable tasks.

### üìÇ Evidence & Logistics
*   **Digital Vault:** Secure, local-first document storage. AI scans PDFs/Images to extract dates and summaries automatically.
*   **Unified Timeline:** A merged view of your legal journal, phone logs, and uploaded documents sorted chronologically.
*   **Expense Tracker:** Logs travel and legal costs for future restitution claims.

### üì¢ Outreach
*   **Comms HQ:** AI email drafter that injects case numbers, context, and specific tone (Firm/Urgent) into correspondence.
*   **Campaign Builder:** Instantly generates and hosts a public SEO-optimized "Bring [Child] Home" website to share with media and law enforcement.

## üõ†Ô∏è Tech Stack

This project is built with a **Privacy-First** architecture.

*   **Frontend:** React 19, TypeScript, Vite
*   **AI:** Google Gemini API (Flash 2.5 & Gemini Live via WebSockets)
*   **Styling:** CSS Variables (Material Design 3 implementation)
*   **Storage (Hybrid):**
    *   **Local:** `localStorage` & `IndexedDB` (for sensitive documents) ensuring data privacy by default.
    *   **Cloud:** Firebase Firestore & Auth (Optional sync for multi-device access).
*   **Utilities:** `jspdf` (PDF generation), `html2canvas` (Image generation).

## üöÄ Getting Started

### Prerequisites
*   Node.js (v18+)
*   Google Gemini API Key
*   Firebase Project (for Cloud Sync features)

### Installation

1.  **Clone the repo**
    ```bash
    git clone https://github.com/yourusername/recovery-hub.git
    cd recovery-hub
    ```

2.  **Install dependencies**
    ```bash
    npm install
    ```

3.  **Environment Setup**
    Create a `.env` file in the root directory:
    ```env
    API_KEY=your_google_gemini_api_key_here
    ```

4.  **Run the development server**
    ```bash
    npm run dev
    ```

## üîí Privacy & Security
*   **Default Mode:** The app operates purely client-side initially. No personal data leaves the device until the user explicitly signs in.
*   **Vault Security:** Documents uploaded to the "Digital Vault" are stored in the browser's `IndexedDB`. They are **not** uploaded to the cloud unless Cloud Sync is active and authorized.
